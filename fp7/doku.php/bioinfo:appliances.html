Bioinformatics Appliances

All these appliances are referenced in the Marketplace and available to use on a StratusLab infrastructure.

  • biodata

The biological databases repository appliance includes the following databases: Swiss-Prot, PROSITE. The appliance can be used to set up your own cloud biological databases repository, and you can add the databases you want with the help of the BioMaj framework. To configure you can use the BioMaj web interface or the command line. To log in with ssh, connect to your VM as root with your SSH authentication key (see general StratusLab documentation for details). To connect to the BioMaj web interface, go to the VM-related URL with a standard web browser: ' http:your.VM.IP/BmajWatcher/ '. Then connect as BioMaj admin with the login biomaj, and configure the databases you want. For more details about the usage of BioMaj, please refer to the official documentation at http://biomaj.genouest.org . This appliance could be used in conjunction with the IDB bioinformatics compute appliance to establish a biological repository for your bioinformatics analyses. For more details, see bioinformatics-related on the StratusLab site.

  • biocompute

The bioinformatics compute appliance comprises the following bioinformatics tools, already installed and available from the command line: BLAST, ClustalW2, Clustal-Omega, FastA, GOR4, HMM, MEME, Muscle, Multalin, PREDATOR, Simpa96, SSearch and R. To log in use ssh with your key and the 'root' account. You have also access to the tools through a web portal, simply connect to your virtual machine with a standard web browser. The appliance can mount the cloud biological database repository (if available) by giving the corresponding contextualization parameters with the stratus-run-instance command. For example to run this appliance on the IBCP cloud, the command looks like: stratus-run-instance –context=“BIO_DB_SERVER=idb-databases.ibcp.fr” Appliance-ID . This appliance can also mount your persistent disk, at boot time or as a hot mount. For more details, see relative documentation on the StratusLab site.

  • ARIA2.3

This appliance is part of the StratusLab bioinformatics usecase TOSCANI (TOwards StruCtural AssignmeNt Improvement). The goal is to improve the determination of protein structures based on Nuclear Magnetic Resonance (NMR) information with the help of a cloud infrastructure based on the StratusLab framework. This concerns the scientific disciplines around (i) molecular and structural biology (determination of biomolecular structures up to atomic resolution) and (ii) bioinformatics, which includes the ensemble of computer algorithms for treating the data from biological systems.This work is done in collaboration between Institut Pasteur (Dr. M. Nilges) and CNRS IBCP (Dr. C. Blanchet).

This appliance includes a fully functionnal version of the ARIA 2.3 software developped in Institut Pasteur. And the current release includes IaaS features to run the computation over several instances. With the usual cloud interfaces, you can launch a bunch of ARIA instances that will be used for a CPU-intensive computation. You connect to one of these virtual machines. This one will be consider as the ARIA master (do not forget to connect with the ForwardingAgent mode activated: 'ssh -A …'). You put in the file '/idb/etc/machines-list' all the IP adresses (one per line) of the ARIA instances you launched. And run the script `aria-clouder' with the ARIA configuration file. An example file, 'dimer_CLUSTER.xml', has been pre-installed in the directory '/idb/aria2.3/examples/dimer'. Go to this directory and run 'aria-clouder dimer_CLUSTER.xml'. This will finish the configuration of the virtual cluster: set up the system services and update the ARIA configuration file to include the computation nodes you previously put in the machines-list file. Finally you simply run the ARIA tool as usual and obtain the results once the calculations are finished. The appliance is built upon Linux CentOS 5.7 distribution.
