User Tutorial

This tutorial covers tasks that typical cloud users will need to do. This tutorial starts from simple, common tasks and progresses through more complex, infrequent tasks. This tutorial assumes that the reader is generally familiar with unix-like systems (principally linux) and has a workstation on which ssh and python are installed.

Controlling a Virtual Machine

The tutorial is divided into the following sections, which the user is recommended to follow in order.

  1. Installation and Setup Covers the installation and setup required on your workstation. At a minimum an ssh client and python are required. Python 2.6 or above is required to authenticate with certificates.
  2. Appliance Repository Discusses the base virtual machine images and appliances that are available.
  3. Machine Lifecycle Describes the commands to push a machine through the standard lifecycle.

Advanced Topics

This section contains information about more advanced tasks that users and administrators may want to accomplish. These topics should also be followed in order as later ones reuse or build on earlier topics.

  1. Creating a Base Image Walks through how to create the ttylinux base image used in the tutorial. This gives a general overview of the steps necessary to create a new base image.
  2. CentOS Base Image Shows how a CentOS base image can be created in a fully automated fashion.
  3. Ubuntu Base Image Describes the creation of a Ubuntu base image. The creation is largely automated but still requires some manual intervention.

Image signature and validation

New in v0.2 In order to create trust between users and system administrators regarding the creation of new instances at a given site, establishing a pedigree and provenance of images important. To this effect, two new user commands are available such that users can sign and verify the manifest describing virtual images.

To sign the manifest, run the following command: 

$ stratus-sign-metadata -h
Usage: usage: stratus-sign-metadata [options] [manifest-file]

Options:
  -h, --help            show this help message and exit
  -v, --verbose         verbose level. Add more to get more details.
  --p12-cert=FILE       PKCS12 (P12) certificate file. Default
                        STRATUSLAB_P12_CERTIFICATE
  --p12-password=PASSWORD
                        PKCS12 (P12) password. Default STRATUSLAB_P12_PASSWORD
  --output=OUTPUTMANIFESTFILE
                        newly signed manifest file (default [manifest-
                        file].sign)

To verify the validity of a manifest, run the following command: 

$ stratus-validate-metadata -h
Usage: usage: stratus-validate-metadata [options] [manifest-file]

Options:
  -h, --help     show this help message and exit
  -v, --verbose  verbose level. Add more to get more details.

As you can see, these commands rely on PKCS12 (P12) certificates.

Future releases of StratusLab will build on these commands to provide a full verification chain, including the location of the image and its hash such that system administrators (and StratusLab execution engine) can ensure that only trusted and untempered images are executed.
